template<typename Callable , typename... Args>
using Voxx::Hetero::Detail::DeviceLauncher = typedef std::conditional_t<mayVary<Args...>, DynamicLauncher<Callable, Args...>, StaticLauncher<Callable, Args...> >

Alias for the type of device callable closure to use.

If the arguments container a non const lvalue reference then they may vary (i.e invoking the callable may set the reference) and hence we need to allocate them on the device and copy them back into the original args after the call. This alias defines the type of device closure depending on the above.
