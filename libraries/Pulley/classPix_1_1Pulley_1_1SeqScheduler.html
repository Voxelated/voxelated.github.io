The sequential scheduler orders threads based on a "ticket" value, which are sequentially created.

Defines parameters for the scheduler.

Enumerator
WaitBits 

WaitBits defines the number of bits being used to store the difference between the max ticket number and the current ticket number.

WaitMask 

WaitMask specifies the mask of the current waiting threads.

The mask sets all of the WaitBits number of bits to 1 to use for ANDing.

MinSpins 

MinSpins defines the minimum number of spins when trying to wait for a ticket.

MaxSpins 

MaxSpins is the maximum number of spins when trying to wait for a ticket.

uint8_t Pix::Pulley::SeqScheduler< AtomicWrapper >::tryWait ( uint32_t  ticketNum,
AtomicType &  spinCutoff,
time_point< ClockType, Duration > *  maxTime = nullptr 
)
inline

Tries to wait for a ticket to be scheduled (released) by the scheduler.

This will spin for spinCutoff spins, if the updateSpinCutoff parameter is false, otherwise it will spin for Traits::MaxSpins if the ticket is not ready to run within the Traits::MaxSpins spins, in this case the spinCutoff will also be updated using the information generated by spinning.

  • If there is a match and the ticket can run, WaitResult::Success is returned.
  • If the ticket is outdated and should have already run, WaitResult::OldTicket is returned.
  • If the ticket is not matched within the amount of time specified by maxTime, the WaitResult::TimedOut is returned.
Parameters
[in]ticketNumThe number of the ticket to wait for.
[in]spinCutoffThe number of spins to try before waiting.
[in]updateSpinCutoffIf the spinCutoff can be updated.
[in]maxTimeThe max number of time which can be spent trying to wait for the ticket.

State defines the current state of the sequencer.

The state uses 32 - WaitBits for the current ticket, and the last WaitBits which recordes the difference between the max wait index and the current ticket.
