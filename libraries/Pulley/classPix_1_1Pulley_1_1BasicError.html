Detailed Description

template<typename SpecializedErrorType>
class Pix::Pulley::BasicError< SpecializedErrorType >

The BasicError CRTP class defines a base class for simple error types.

A basic error is one which wraps a set of errors, where an error in the set has a unique error code within the set (Logic, Runtime, System, for example, would be basic errors), and which want to specify the location of the error (the function in which the error occurred).

The BasicError class uses a BasicFormatter to format of the logged error. See the BasicFormatter class for how it prints error messages. The BasicFormatter is CRTP specialization of a BaseFormatter, and any error types derived from this one must provide a logImpl() method with the following signature:

template <typename FormatterType>
void logImpl(OStream& os, Formatter<FormatterType>& formatter) const;

The implementation can then use the API defined the by BaseFormatter type to log a message. By default, a BlockFormatter is used to format the error messages.

The constructor takes an optional parameter for extra information related to the error, which makes debugging considerably more simple when extra information is added. For example, in the handler for a LogicError which specifies out of range access, the index used when the out of range access occurs can be added as extra information. Make use of this feature where possible.
