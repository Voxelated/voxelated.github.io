Detailed Description

The TypedError class is a simple, fast error class which can include error context, and ensures that any errors are checked before destruction.

A TypedError instance wraps a TypedErrorInfoBase pointer, which is used to identify the type of the error. A failed error state sets the pointer to a subclass of TypedErrorInfoBase, while a success state sets the pointer to a null pointer.

The class ensures that errors are checked by using the 'Checked' flag, which is be removed at compile time for release code, making the class extremely lightweight. When the flag is not disabled, a runtime check is performed before destruction to state that the error was not checked.

Invoking the conversion to bool will set the 'Checked' flag:

TypedError exampleFunc(...);
// Conversion to bool sets checked flag:
if (auto err = exampleFunc(...))
return err; // Error state: we return the error, it's handled elsewhere

The 'Checked' flag makes sure that we cannot just call a function which returns a TypedError. For example:

auto err = exampleFunc(...);
// or
exampleFunc(...);

The above will trigger a runtime error (when checking is enabled), because the 'Checked' flag is not set when the TypedError is destructed. This happens regardless of the state of the TypedError (success or failure)!

For the case where there is an error failure, there are three functions which can be used to handle the errors: logTypedErrors, catchTypedErrors, and catchAllTypedErrors. These need to have handlers for each of the specific error typed errors to catch (the lambda handler must take a unique_ptr<T> where T is the specific error to catch):

class CustomErrorInfo : public TypedErrorInfo<CustomErrorInfo> {
// Custom error information ...
};
TypedError example(...) {
return makeTypedError<CustomErrorInfo>(...);
}
// Some logger:
Logger logger;
// If there is failure here, CustomErrorInfo is returned.
auto err = example();
auto newErr = checkTypedErrors(std::move(err),
[&] (std::unique_ptr<CustomErrorInfo> ce) {
// Handle the error -- just by logging it.
ce->log(logger.oStream);
// Return success error:
return TypedError();
},
[&] (std::unique_ptr<FatalError> fe) {
fe->log(logger.oStream);
logger.flush();
// Terminate on fatal error:
std::terminate;
}
);

Note: The handlers also return TypedErrors, since it's possible that they might also incurr and error.

The catchAllTypedErrors is the same as catchTypedErrors, except that it has a void return type. This implies that it be able to handle all possible errors. It prevents errors from being pushed back up the call stack.

Note 2: The syntax in some cases may be confusing when evaluating a TypedError as a boolean. Because the class is an error class, its evaluation as a boolean evaluates to true when it is an error. Testing for the success case can then be confusing:

  • if (TypedError()) {...}: True when TypedError is an error which needs to be handled.
  • if (!TypedError()) {...}: True when TypedError is an error which does not need to be handled i.e is the success case.

The catchTypedErrors function needs to be able to set the Checked flag.

Any errors which are not handled are re-concatenated and returned.

As this function returns a TypedError, the result needs to also be checked (for example for a case when there was no handler for one or more of the errors). If all errors must be caught, then use catchAllTypedErrors. Example usage is as follows:

class CustomError : public TypedErrorInfo<CustomError> {};
auto e = makeTypedError<CustomError>();
auto results = catchTypedErrors(std::move(e),
[] (std::unique_ptr<CustomError> ce) {
// Do handling ...
return TypedError();
},
[] (std::unique_ptr<OtherError> oe) {
// Do handling ...
return TypedError();
}
